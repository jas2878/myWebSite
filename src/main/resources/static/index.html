마커

지도의 해당 위치에 아이콘을 표시하고 상세정보를 텍스트, 이미지, 동영상의 형태로 제공할 수 있는 기능을 등록할 수 있습니다.

지역 선택

• 지역이 동일한 사용자가 기능을 실행할 수 있습니다. 예를 들어 기능의 지역 선택 값이 부산광역시 라면, 사용자 지역이 부산광역시 인 사용자에게만 기능이 표시됩니다. 전체지역을 선택하면 모든 사용자가 기능을 실행할 수 있습니다.

팝업 종류

• 지도에서 마커를 클릭했을 때 제공하는 정보의 유형입니다. 텍스트는 표 형태로 제공하며, 이미지와 동영상은 name과 url 필드명을 입력해야 합니다.

데이터 종류

• 기능에 대한 데이터는 파일과 API의 2가지 방식으로 입력될 수 있습니다. API를 선택할 경우 연동할 API 인증키와 요청 값을 입력 하십시오.

필드 매핑

• 지도 매핑은 좌표 또는 주소로 처리됩니다. 팝업 종류가 테이블인 경우 정보 제공 필드를 입력합니다. 실제 필드명과 표시할 이름을 각각 입력합니다.

연관 기능 지정

• 선택한 솔루션(지진, 침수, 미세먼지)의 다른 기능을 팝업 창에 함께 목록으로 보여줄 수 있습니다.

SHP

선 또는 면으로 된 지리정보를 지도에 시각화 할 수 있는 기능입니다. 파일 형식은 SHP 및 관련 파일이며, 시각화 스타일을 설정할 수 있습니다.

파일 업로드

• SHP 파일 속성정보를 참고하여 레이어로 처리할 변수를 선택합니다. 단색 또는 복수색으로 설정할 수 있습니다. • 복수형의 경우, 변수 특성에 따라 분류형과 구간형을 선택합니다. - 구간인 경우 범례개수를 입력합니다. 각 구간의 시작과 끝 값을 직접 입력하여 바꿀 수 있습니다. - 색상 설정에서 각 색상을 클릭하여 원하는 색상으로 변경합니다. • 단색의 경우, 패턴을 지정할 수 있습니다. 패턴은 선의 간격과 굵기, 모양을 선택할 수 있습니다. • 색상 설정 또는 패턴색에서 색상을 클릭하여 원하는 색으로 변경합니다.

히트맵

좌표에 해당하는 숫자 값을 구간별 레이어 설정을 통해 히트맵의 형태로 시각화 하는 기능입니다.

필드 설정

• 쉼표로 분리된 UTF-8 형식의 CSV 파일을 업로드합니다. • 업로드 파일에 들어있는 필드 중 위도, 경도, 히트맵으로 처리할 변수를 선택합니다. • 구간의 개수를 입력하거나 선택합니다. 필요한 경우 개별 구간의 시작과 끝 값을 변경할 수 있습니다.

시간당 침수 시나리오 구축

• 시간당 70, 90, 120mm/h 침수 시나리오 구축 • 시/군/구/분구별 구역 및 범위 설정 가능 • SWMM, ITZI 등 유역 모델링 엔진 기반 국내 실정에 맞는 침수 예방 프로그램 도입

시나리오관리

솔루션 시나리오별 최적대피로 안내

• 지진, 미세먼지 – 집결지 출발 대피소 안내 • 침수 – 침수 지역 침수심 기준 이상 지역 대피

대피로 데이터

• 도로명주소 노드링크 데이터 구축 • 표준노드링크 데이터 구축 • 도로명주소+표준노드링크 혼합(테스트 중)

최적대피로 안내

• 대피로 계산 시 대피 지역 설정 • 일정 수준 이상인 침수심이 발생한 지역만 대피하도록 설정. • 침수 지역 내에 포함된 노드를 폴리곤에서 1개만 추출 하여 출발지로 설정하며, 직선거리가 가까우며 침수 되지 않은 대피소 3개에 대하여 Dijksatra 및 A* 알고리즘에 따른 경로값을 계산

모바일 안전대피로 앱

• 크롬 PWA(Progressive Web Application) 방식의 웹앱 구현 • 목표 이용자: 일반 시민 • 현재 위치정보 접근 허용 시 현위치 기준 대피로 제공 • 검색 주소 지오 코딩 처리 후 대피로 제공 • 대피로 API 처리 • 현위치/검색주소 위치 기준 요청 • 출발지 결정 후 최단거리 대피로 산출 • 도착지, 대피경로, 침수예상지역 데이터 제공 • 현재 위치 침수 예측 정보와 유사한 내수침수 시나리오 침수 지역 표시 • 0 초과 50 이하 - 50 mm 시나리오 침수 지역, 50 초과 105 이하 – 105 mm 시나리오 침수 지역, 105 초과 – 150 mm 시나리오 침수 지역

도로명주소 활용 도로 네트워크 구축

• 도로명주소 • 표준노드링크 • 원래 분리되었던 노드를 통합하여 새로운 경로 생성 가능 • 노드가 거리가 아래 그림은 20M 내에 있는 3개의 노드를 1개의 표준노드링크 노드ID로 통합한 후 지도와 길찾기 쿼리 예시 • 도로명주소와 표준로드링크가 연결이 되어 서로 길찾기가 가능해짐
